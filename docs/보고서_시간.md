# 프로젝트 진행 보고서
**작성 일시**: 2025-11-28 11:20

## 1. 개요
본 보고서는 NeuroNova 프로젝트의 개발 환경 구축 및 Docker 통합 작업에 대한 진행 상황을 정리합니다.

## 2. 주요 작업 완료 내역

### 2.1 환경 설정 자동화 (Environment Setup Automation)
- **목표**: 개발자별 환경 변수 설정의 번거로움 해소 및 중앙 관리.
- **결과**:
    - `docs/setup_env.bat` (Windows) 및 `docs/setup_env.sh` (Linux/Mac) 스크립트 고도화.
    - `docs/` 폴더 내에 `backend.env`, `frontend.env` 파일이 존재할 경우 이를 우선적으로 적용하도록 로직 개선.
    - 커스텀 파일이 없을 경우 기존대로 `.env.example`을 사용하도록 Fallback 처리.

### 2.2 Docker 통합 (Docker Integration)
- **목표**: 개발 환경의 통일성 확보 및 구글 VM 배포 준비.
- **결과**:
    - **Backend (Django)**: `backend/django_main/Dockerfile` 생성 (Python 3.10 기반).
    - **Backend (Flask)**: `backend/flask_inference/Dockerfile` 생성 및 플레이스홀더 앱(`app.py`) 구현.
    - **Frontend (React)**: `frontend/react_web/Dockerfile` 생성 (Node.js 빌드 + Nginx 서빙).
    - **Orchestration**: `docker-compose.yml`을 통해 Django, Flask, React, PostgreSQL, Redis, Orthanc 6개 서비스를 통합 관리하도록 구성.

### 2.3 구성 검증 (Configuration Check)
- **React**: `src/utils/config.js`가 `VITE_` 접두사가 붙은 환경 변수를 사용하도록 올바르게 설정됨을 확인.
- **Flutter**: `lib/core/config/app_config.dart`가 `String.fromEnvironment`를 사용하여 빌드 시 환경 변수를 주입받도록 설정됨을 확인.

## 3. 현재 상태 및 이슈
- **Docker Desktop**: 현재 작업 환경에 Docker Desktop이 실행되고 있지 않아 빌드 테스트(`docker-compose build`)는 보류되었습니다.
- **파일 준비**: 배포 및 실행에 필요한 모든 설정 파일(Dockerfile, docker-compose.yml, setup scripts)은 준비 완료되었습니다.

## 4. 향후 계획
1.  **Docker 실행**: Docker Desktop 설치 및 실행 후 `docker-compose up --build`로 전체 서비스 가동 확인.
2.  **구글 VM 배포**: `backend` 관련 서비스(Django, Flask, DB, Redis, Orthanc)를 클라우드 환경에 배포.
3.  **기능 개발**: API 연동 및 추가 기능 구현 진행.
